# User Authentication & Management API

A lightweight Node.js/Express backend that implements a structured user schema using Mongoose and MongoDB.

## 📋 User Schema Specifications

The database layer strictly enforces the following schema architecture:

| Field Name  | Type     | Constraints / Validations                            |
| :---------- | :------- | :--------------------------------------------------- |
| `_id`       | ObjectId | Primary key, auto-generated by MongoDB               |
| `name`      | String   | Required, trimmed of whitespace                      |
| `email`     | String   | Required, unique, trimmed, regex email validated     |
| `password`  | String   | Required, minimum length: 6                          |
| `role`      | String   | Enum: `["customer", "admin"]`, Default: `"customer"` |
| `createdAt` | Date     | Auto-generated timestamp                             |
| `updatedAt` | Date     | Auto-updated timestamp                               |

# Project Dependency Overview

This project utilizes a modern Node.js and Express backend ecosystem. Below is a breakdown of the core packages and tools driving the application, along with their specific roles in development, database management, security, and routing.

## Development & Environment

- **`nodemon`**
  Runs the development server efficiently by automatically restarting the Node application whenever file changes are detected in the directory.
- **`dotenv`**
  Safely boots up your secret MongoDB connection credentials and other sensitive environment variables from a `.env` file, keeping them out of source control.

---

## Database Management

- **`mongoose`**
  Connects the app securely to your MongoDB database clusters, providing a straight-forward, schema-based solution to model your application data.

---

## Routing & Cross-Origin Requests

- **`express`**
  The core web framework that handles server logic and routing, opening up API routes such as `POST /api/register`.
- **`cors`**
  Cross-Origin Resource Sharing (CORS) middleware that configures the server's security policies, allowing your React or Vue frontend application to hit backend routes seamlessly.

---

## Security & Authentication

- **`bcrypt`**
  Secures user data by hashing the new user's password using a strong cryptographic algorithm before `mongoose` stores it in the database.
- **`jsonwebtoken`**
  Issues a secure, digitally signed token (JWT) when users later log in, keeping them authenticated across subsequent requests without needing to re-enter credentials.

---

### Getting Started

To install all the dependencies listed above in your local project directory, run:

npm install --save-dev nodemon
